O Sistema Operacional GNU/Linux

O Sistema Operacional GNU/Linux

Richard Stallman, criador do projeto GNU

  • Em 1983 Richard Stallman começou o Projeto GNU;
  • O objetivo era criar um Sistema Operacional compatível com Unix porém, diferente deste, o GNU seria totalmente Open Source;
  • No começo dos anos 90 o Sistema estava quase pronto, faltava as funcionalidades de baixo nível como drivers e o kernel (núcleo) do sistema.

O Sistema Operacional GNU/Linux

Linus Torvalds, criador do kernel Linux

  • Em 1991, Linus Torvald, enquanto cursava Ciência da Computação na Universidade de Helsinki começou um projeto que mais tarde viria a se tornar o kernel Linux;
  • Ele usou o gcc, o GNU C Compiler,que até hoje é a opçãao utilizada para compilar o kernel Linux;
  • Em 1992 Linus mudou a licença do kernel para GPL, o que possibilitou a integração com o GNU de forma a manter o Sistema Open Source;

O Sistema Operacional GNU/Linux

  • Hoje o kernel Linux é mantido por várias empresas e pode ser encontrado em: http://www.kernel.org

Distribuições

Distribuições

  • Existe uma grande variedade de sistemas operacionais baseados em GNU/Linux, cada um com um objetivo específico;

  • Como o kernel linux dá suporte a várias arquiteturas de processadores, existem sistemas linux voltados para servidores, para supercomputadores, sistemas embarcados/dispositivos IoT, além dos sistemas para uso pessoal;

  • As versões do sistema operacional são chamadas de distribuição (informalmente chamada de distro) e cada uma possui seu conjunto de programas padrão, gerenciador de pacotes (aplicações) e seus repositórios de programas.

Distribuições

Apesar do grande número de distribuições, podemos dividir elas por "famílias" nomeadas pela distribuição na qual elas são baseadas.

Essas famílias são:

  • Família Debian
  • Família Slackware
  • Família Redhat
  • Família Enoch/Gentoo
  • Família Archlinux

Também podemos classificar as distribuições em dois tipos de acordo com a sua forma de atualização:

  • Long Term Support (LTS)
  • Rolling Release

As distribuições LTS lançam novas versões mantendo atualizações de versões antigas, semelhante ao que ocorre no Windows.

As distribuições Rolling Release lançam atualizações contínuas de modo que a cada atualização você sempre terá a versão mais recente do sistema.

Ambientes Desktop

  • Um ambiente desktop é composto por um gerenciador de janelas, widgets e um conjunto de programas;

  • O gerenciador de janelas é responsável pelo posicionamento e aparência das janelas;

  • Os ambientes desktop também fornecem widgets, componentes prontos para desenvolvedores usarem como botões, caixas de diálogos, etc;

  • Mesmo que a maioria das distribuições venham com um ambiente desktop padrão, é possível instalar qualquer ambiente em qualquer distribuição.

Gerenciadores de Exibição

Gerenciadores de exibição são programas que funcionam como um frontend para inicialização do sistema.

Fornecem uma interface para login de usuário e seleção de ambiente desktop.

Gerenciadores de Pacotes

Gerenciadores de Pacotes

Os gerenciadores de pacote organizam os pacotes instalados no sistema e são responsáveis por:

  • Verificar checksums para ter certeza de que o pacote foi baixado sem erros;

  • Verificar a assinatura digital para autenticar a origem dos pacotes;

  • Atualizar um sistema e corrigir bugs (principalmente erros de segurança);

  • Agrupar pacotes de acordo com suas funcionalidades para reduzir confusões;

  • Gerir dependências para ter certeza de que um pacote só será instalado se todas as suas dependências também estiverem, o que resolveu o problema conhecido como "Inferno de dependências".

Gerenciadores de Pacotes

Mesmo com instaladores gráficos, a maioria dos usuários prefere fazer instalações via terminal por ser mais rápido. Abaixo um exemplo de como instalar o openssh usando o gerenciador de pacotes do Ubuntu (apt):
bash
$ sudo apt install openssh

Gerenciadores de Pacotes

Exemplo de como desinstalar o openssh usando o gerenciador de pacotes do Ubuntu (apt):
bash
$ sudo apt remove openssh

Gerenciadores de Pacotes

Exemplo de como atualizar a base de dados do gerenciador de pacotes do Ubuntu (apt):
bash
$ sudo apt update

Gerenciadores de Pacotes

Exemplo de como aplicar (instalar) as atualizações disponíveis com o gerenciador de pacotes do Ubuntu (apt):
bash
$ sudo apt upgrade

Gerenciadores de Pacotes

Exemplo de como remover pacotes órfãos (pacotes que foram instalados como dependências mas que já não estão mais em uso):
bash
$ sudo apt autoremove

Gerenciadores de Pacotes

Exemplo de como buscar um pacote:
bash
$ sudo apt search <nome_do_pacote>
